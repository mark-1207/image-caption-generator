# 优化报告页面开发计划

## 页面定位

优化报告页面是内容改写工坊的第4步（最后一个步骤），用于对AI生成的各平台内容进行智能质检分析，依据评定规则分析内容质量，给出优化建议，并基于优化建议对生成内容进行二次生成，确保内容质量优质。

---

- 左侧：生成的文章内容（可编辑）

## 二、功能模块详细设计

### 2.1 平台切换功能

- 顶部tab切换：公众号 | 小红书 | 抖音
- 切换平台后刷新对应的质检数据
- 当前选中的平台高亮显示

### 2.2 六维度雷达图

使用 Recharts 库绘制雷达图

**六维度定义**：
| 维度 | 公众号标准 | 小红书标准 | 抖音标准 |
|------|-----------|-----------|---------|
| 标题吸引力 | 13-25字，关键词前置 | 20字内，情绪词+关键词 | 15字内，悬念/冲突 |
| 开头留存力 | 前300字留存 | 前50字留存 | 前3秒完播 |
| 内容价值度 | 知识增量+认知颠覆 | 实用干货+种草价值 | 信息密度+节奏 |
| 情绪感染力 | 共鸣+深度 | 真实+温暖 | 强烈+即时 |
| 传播设计度 | 金句数+转发点 | 互动引导+收藏点 | 点赞引导+评论点 |
| 排版美观度 | 段落+重点标记 | Emoji+分段 | 字幕+画面 |

**雷达图样式**：
- 填充色：平台对应颜色（公众号蓝色、小红书粉色、抖音青色）
- 背景网格：浅灰色
- 分数显示在图表中心或旁边

### 2.3 整体评分展示

- 显示综合评分（0-10分）
- 评分等级标签：
  - 9-10分：⭐⭐⭐ 爆款潜质
  - 7-8分：⭐⭐ 优质内容
  - 5-6分：⭐ 合格内容
  - <5分：⚠️ 需优化

### 2.4 质检清单

**展示形式**：列表形式，每个检查项显示：
- 状态图标：✅ 通过 / ❌ 未通过
- 检查项名称
- 一句话理由

**检查项列表**（示例）：
```
✅ 标题长度合适 - 字数在13-25字之间
✅ 开头有钩子 - 前50字包含悬念或痛点
❌ 缺少金句引用 - 正文未引用金句
✅ 结尾有CTA - 包含行动号召
✅ 信息密度足够 - 包含2个以上知识点
✅ 情感表达恰当 - 情绪浓度适中
❌ 互动引导不足 - 未包含评论引导
✅ 排版规范 - 段落清晰，重点标记明显
```

### 2.5 优化建议

**展示形式**：可折叠的列表

**每个建议包含**：
- 序号
- 建议内容描述
- 涉及的具体位置（如：第2段、结尾）

**示例**：
```
优化建议（3条）

1. 📝 在第2段开头加入一个金句引用
   位置：正文第2段
   建议添加："xxx" - 作者

2. 💬 在结尾增加互动引导
   位置：结尾部分
   建议添加：互动问题或投票

3. 🎯 标题可增加热点词汇
   位置：标题
   建议：加入2026、职场等热词
```

### 2.6 一键优化功能

**功能说明**：
- 点击"一键优化"时，仅对**当前选中的平台**内容进行优化
- 优化的依据是**优化报告的建议**，优先处理做得不好的部分（评分低的维度）
- 优化后展示对比浮层，让用户选择

**点击"一键优化"后的流程**：
```
1. 用户点击"一键优化"按钮
2. 显示加载状态（AI正在根据优化建议生成新内容...）
3. 生成优化后的内容
4. 弹出对比浮层
```

### 2.7 优化对比浮层

**浮层布局**（左右对比）：
```
┌─────────────────────────────────────────────────────────────┐
│  优化对比                                           [×]      │
├─────────────────────────────────────────────────────────────┤
│  当前平台：公众号                                              │
│                                                             │
│  ┌─────────────────────┐  ┌─────────────────────────────┐  │
│  │  优化前             │  │  优化后                     │  │
│  │                     │  │  ⭐ 基于以下建议优化         │  │
│  │  [标题内容]         │  │  • 建议1...                 │  │
│  │                     │  │  • 建议2...                 │  │
│  │  [正文内容]         │  │                             │  │
│  │  ...                │  │  [标题内容]                 │  │
│  │                     │  │                             │  │
│  │                     │  │  [正文内容]                 │  │
│  │                     │  │  ...                        │  │
│  └─────────────────────┘  └─────────────────────────────┘  │
│                                                             │
│              [使用优化前]              [使用优化后 ✓]        │
└─────────────────────────────────────────────────────────────┘
```

**用户选择后**：
- 选择"使用优化后"：更新左侧内容为优化后的版本，更新右侧优化报告
- 选择"使用优化前"：保持原内容不变
- 选择后可点击"导出"，流程结束

### 2.7 底部操作区

- **上一步**：返回内容创作页面
- **一键优化**：触发AI优化（可点击）
- **确认完成**：完成整个流程，可选择：
  - 再次编辑
  - 导出内容
  - 新建项目

## 三、数据结构设计

### 3.1 质检报告数据结构

```typescript
interface QualityReport {
  // 整体评分
  overallScore: number;           // 0-10
  grade: 'excellent' | 'good' | 'average' | 'poor';

  // 六维度评分
  dimensions: {
    titleAppeal: number;           // 标题吸引力
    openingRetention: number;     // 开头留存力
    contentValue: number;         // 内容价值度
    emotionInfluence: number;     // 情绪感染力
    viralDesign: number;          // 传播设计度
    layoutBeauty: number;         // 排版美观度
  };

  // 质检清单
  checklist: Array<{
    id: string;
    item: string;                  // 检查项名称
    passed: boolean;              // 是否通过
    reason: string;               // 理由说明
    position?: string;            // 涉及位置（可选）
  }>;

  // 优化建议（与质检清单关联）
  optimizationSuggestions: Array<{
    id: string;
    content: string;              // 建议内容
    position?: string;            // 涉及位置
    priority: 'high' | 'medium' | 'low';  // 优先级
    relatedCheckId?: string;      // 关联的质检项ID
  }>;
}
```

### 3.2 每个平台的完整数据结构

```typescript
interface PlatformContent {
  platform: 'gzh' | 'xhs' | 'douyin';
  title: string;                  // 当前标题
  content: string;                 // 当前正文
  coverPrompt: string;             // 封面提示词
  qualityReport: QualityReport;    // 质检报告
  isOptimized: boolean;            // 是否已优化过
  optimizedVersion?: {             // 优化后的版本（待用户选择）
    title: string;
    content: string;
    qualityReport: QualityReport;
  };
}
```

### 3.3 页面 Props 设计

```typescript
interface OptimizationReportPageProps {
  // 从上一步传入的数据 - 各平台生成的内容
  generationResult: {
    platforms: {
      gzh: PlatformContent;
      xhs: PlatformContent;
      douyin: PlatformContent;
    };
    selectedPlatforms: string[];   // 用户选择需要优化的平台
  };

  // 回调函数
  onBack: () => void;             // 返回上一步
  onExport: (platform: string) => void;  // 导出指定平台内容
  onComplete: () => void;         // 完成整个流程
}
```

## 四、交互流程

### 4.1 页面加载流程

```
1. 用户从内容创作页面点击"下一步"
2. 显示加载状态（AI正在生成质检报告）
3. 获取当前平台的生成内容
4. 调用AI质检接口（或使用模拟数据）
5. 渲染质检报告页面（默认显示第一个选中的平台）
```

### 4.2 平台切换流程

```
1. 用户点击平台Tab（如从公众号切换到小红书）
2. 加载该平台的内容和质检数据
3. 左侧更新为该平台的生成内容
4. 右侧更新为该平台的优化报告
```

### 4.3 一键优化流程

```
1. 用户点击"一键优化"按钮
2. 仅对当前选中的平台进行优化
3. 根据优化报告中的建议（优先处理评分低的维度）
4. 生成优化后的新内容
5. 弹出对比浮层，展示优化前后的内容
6. 用户选择使用"优化前"或"优化后"
7. 更新页面内容（如果选择了优化后）
```
