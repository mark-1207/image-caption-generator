# 微博热搜产品创意分析技能

## 概述

本技能用于分析微博热搜榜单，从热点事件中挖掘产品创意机会。通过API获取热搜数据、搜索引擎收集背景资讯、AI分析生成产品创意，最终输出可视化的HTML分析报告。

## 核心功能

1. **热搜数据抓取** - 通过配置的API获取微博实时热搜榜单
2. **热点背景搜索** - 为每个热搜话题搜索相关新闻和背景信息
3. **产品创意生成** - 基于热点分析，AI生成产品创意方案
4. **HTML报告生成** - 输出结构化的分析报告，含评分和可视化

## 使用场景

- 产品经理寻找新功能灵感
- 创业者挖掘市场机会
- 内容创作者获取选题素材
- 投资人了解当下热点趋势

## 工作流程

### 阶段1：获取热搜榜单

**输入**：
- 用户提供的微博热搜API地址（URL）

**操作**：
1. 调用指定API获取热搜数据
2. 解析返回数据，提取热搜话题、排名、热度值等
3. 筛选出需要分析的重点话题（可按热度排序选取前10-20条）

**输出格式**：
```json
[
  {
    "rank": 1,
    "topic": "热搜话题名称",
    "heat": 热度值,
    "url": "话题链接"
  }
]
```

### 阶段2：搜索热点背景

**输入**：热搜话题列表

**操作**：
对每个热搜话题使用WebSearch工具搜索：
1. 新闻报道和最新进展
2. 事件背景和起因
3. 社交媒体讨论和网民观点
4. 相关话题和延伸信息

**搜索策略**：
- 使用话题关键词进行搜索
- 优先选择权威媒体和主流新闻源
- 收集不同角度的观点和信息

### 阶段3：AI产品创意分析

**输入**：热点背景信息

**分析维度**：
- **有趣度 (80%权重)**：话题的讨论热度、传播性、娱乐性、话题延展性
- **有用度 (20%权重)**：话题的商业价值、用户需求真实性、市场空白

**评分标准**：
| 评分区间 | 等级 | 说明 |
|---------|------|------|
| >80分 | 优秀 | 高度推荐的产品创意机会 |
| 60-80分 | 良好 | 有潜力的产品创意 |
| <60分 | 一般 | 创意空间有限 |

**产品创意结构**：
```json
{
  "product_name": "产品名称",
  "core_features": ["核心功能1", "核心功能2", "核心功能3"],
  "target_users": "目标用户描述",
  "fun_score": 有趣度分数,
  "usefulness_score": "有用度分数",
  "total_score": 综合评分,
  "analysis_rationale": "分析依据"
}
```

## 输出规范

### HTML报告要求

1. **编码**：UTF-8
2. **样式**：简洁专业，包含响应式设计
3. **交互**：可折叠展开详情，方便浏览
4. **图表**：可使用CSS实现简单可视化

### 评分计算

```
综合评分 = 有趣度 × 0.8 + 有用度 × 0.2
```

### 有趣度评分标准（0-100分）

| 维度 | 评分要点 |
|------|---------|
| 话题热度 | 搜索量、讨论量、传播速度 |
| 娱乐性 | 话题的趣味性、可玩性 |
| 延展性 | 话题的二次创作空间 |
| 情感共鸣 | 能否引发用户情感反应 |
| 时效性 | 话题的持久性和更新频率 |

### 有用度评分标准（0-100分）

| 维度 | 评分要点 |
|------|---------|
| 商业价值 | 变现能力和市场规模 |
| 用户需求 | 需求的真实性和紧迫性 |
| 市场空白 | 竞品情况和进入门槛 |
| 技术可行性 | 实现难度和技术要求 |
| 持续性 | 需求的长期存在性 |

## 注意事项

1. **API配置**：确保用户提供了有效的微博热搜API地址
2. **搜索质量**：搜索时使用准确的关键词，获取多角度信息
3. **创意可行性**：分析时要考虑实际的商业和技术可行性
4. **版权合规**：分析报告仅供学习和参考
5. **数据时效**：热搜具有时效性，分析结果基于当前时间点

## 限制说明

- 本技能依赖外部API获取热搜数据
- AI创意分析基于现有信息和通用知识
- 生成的创意需要进一步的市场验证
- 不涉及具体的代码实现和开发
- 报告仅供参考，不构成投资建议
